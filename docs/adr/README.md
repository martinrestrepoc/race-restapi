# Architecture Decision Records

This directory contains the accepted architecture decisions for the EIA Racing
System. An ADR records a decision and its consequences; it does not by itself mean
that the corresponding implementation already exists.

| ADR | Decision |
| --- | --- |
| [0001](0001-domain-lifecycles-and-deletion.md) | Domain lifecycles and deletion semantics |
| [0002](0002-results-standings-and-audit.md) | Results, standings, and audit behavior |
| [0003](0003-keycloak-and-local-identity.md) | Keycloak roles and local identity |
| [0004](0004-identifiers-measurements-and-time.md) | Identifiers, measurements, and time values |
| [0005](0005-runtime-infrastructure-and-testing.md) | Runtime, infrastructure, and testing |
| [0006](0006-frontend-and-repository-layout.md) | Frontend stack and repository layout |

When a decision changes, add a new ADR that supersedes the old one rather than
rewriting historical context.
